# Frontend

前端约束应强调可验证性和可观察性。

## 约束

待技术栈确定后补充具体框架约束。以下为通用原则：

### 结构

- 按功能/领域组织组件，不按类型（components/, hooks/, utils/）
- 页面组件只做编排，不含业务逻辑
- 共享组件放入 `shared/` 或设计系统包

### 状态管理

- 组件内状态优先（useState / 局部状态）
- 跨组件共享状态使用显式边界（Context, Store, Signal）
- 服务端状态与客户端状态分离
- 不可变更新模式：创建新对象，不修改旧对象

### 可验证性

- 关键用户流有 E2E 测试覆盖
- 交互组件有单元测试
- 可访问性基线: 语义化 HTML, ARIA 标签, 键盘导航
- 性能基线: LCP < 2.5s, FID < 100ms, CLS < 0.1

### Harness Evaluator 如何评审前端

Evaluator agent 验收前端 sprint 时关注：

1. 验收标准中的 UI 行为是否可通过测试验证
2. 组件是否遵循不可变模式
3. 边界输入（用户输入、API 响应）是否有校验
4. 无硬编码值（颜色、尺寸、文案用常量或配置）
