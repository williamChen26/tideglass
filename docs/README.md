# Docs Index

`docs/` 是这个仓库的记录系统。

如果一个智能体无法在仓库中发现某条信息，那么对它来说，这条信息就不存在。

## 目录

- [`docs/design-docs/index.md`](docs/design-docs/index.md): 设计文档目录与核心理念。
- [`docs/product-specs/index.md`](docs/product-specs/index.md): 产品规格目录与验收入口。
- [`docs/exec-plans/index.md`](docs/exec-plans/index.md): 执行计划、完成记录与技术债。
- [`docs/generated/README.md`](docs/generated/README.md): 生成型工件说明。
- [`docs/references/index.md`](docs/references/index.md): 外部参考与内部摘录。

## 维护规则

1. 新增长期知识时，优先放入 `docs/` 而不是聊天记录。
2. 新建文档时，要把它链回对应的 `index.md`。
3. 已完成执行计划要移动到 `completed/`，不要长期堆在 `active/`。
4. 生成型文档要注明来源、生成方式和刷新时机。
