# Design

设计目标是帮助智能体和人类快速形成一致判断。

## 设计原则

1. **约束是速度的前提**: 严格的边界和可预测的结构让智能体高效工作，而不是拖慢速度。
2. **不变量优于微管理**: 强制不变量（边界校验、依赖方向），但允许实现自由。
3. **机械执行优于口头约定**: 能变成 lint/测试的规则，不应该只写在文档里。
4. **智能体可读性优先**: 仓库结构优先为智能体的认知模型优化，其次是人类美学。

## Harness 如何影响设计

Harness 架构引入了一个关键设计约束：**每个设计决策都编码了一个关于模型能力不足的假设**。

- Planner 存在是因为模型不能从一句话直接产出高质量代码
- Evaluator 存在是因为模型不能可靠地自我批评
- 3 轮迭代上限是因为如果 3 轮解决不了，说明方法有问题

这些假设值得定期压力测试。当模型能力提升时，架构应该简化而不是固化。

## 权衡偏好

| 当面临 | 偏向 | 理由 |
|--------|------|------|
| 简单 vs 完整 | 先简单，按需增加 | 最简方案优先 |
| 通用 vs 专用 | 先专用，重复 3 次再抽象 | 避免过早抽象 |
| 文档 vs 代码约束 | 代码约束 | 机械执行不会遗忘 |
| 智能体自主 vs 人类确认 | 低风险自主、高风险确认 | 平衡效率与安全 |
